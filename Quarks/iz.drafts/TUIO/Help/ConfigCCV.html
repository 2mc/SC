ConfigCCV


Utility for configuring 'Community Core Vision.app' by changing entries in the config file: 'data/config.xml'. 

File data/config.xml is included in the current release of CCV. (CCV 1.2) and stores the current configuration of a ccv session. You need to restart ccv for changes to take effect. 


Usage 


*setConfigPath


Choose the path of config.xml where the video path is stored for ccv.


ConfigCCV.setConfigPath; // choose the path of config.xml where the video path is stored for ccv


*setConfigPath


Choose the path of the video file to be used by ccv


ConfigCCV.setVideoPath; // choose the path of the video file to be used by ccv


Note: You should quit CCV before using this method, because CCV always stores its current configuration to config.xml when it quits, so if you quit CCV after changing the video path, CCV will overwrite your changes. 


*openConfigFile


Open the config.xml file specified by the user, using unix command: "open <configPath>". 

Note: This will open the config.xml file using the default application set for .xml. 


ConfigCCV.openConfigFile;


Other

Intermediate checking methods (for development) are: 

ConfigCCV.getVideoPath;



This opens a dialog file for selecting the path for the video file to be used by Community Core Vision.  If ConfigCCV does not know the location of the config file of Community Core Vision, it will first open another dialog to get the path of the config.xml file from the user.
