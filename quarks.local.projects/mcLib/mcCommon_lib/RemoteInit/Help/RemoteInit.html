RemoteInit allow a class (typically with config files) to be inited by another 'remote' class


RomoteInit supports loadig of a configFile per Class with a pre and post delay

  This service class assusmes that serviced Classes implement as defaults:

  classvars <relativeDir, <configFile

  Also, there are callbacks to the methods:

  *preLoadInit{|who, *args| }, *postLoadInit{|dir| }, *failLoadInit{|dir| }    

  If a remote class (other than the serviced Class) was calling for a 'romoteInit', 

  the caller gets a final callback to: updRemoteInit{|iClass, state| }

  states are: 0 -> inited by registerant, 1 -> inited by sombody else, 2 -> failed

   

RemoteInit.classes.keysDo{|clss, more| [clss, RemoteInit.classes[clss] ].postln; "\n".post}; ""

RemoteInit.callers.keysDo{|calr, classes| [calr, RemoteInit.callers[calr] ].postln; "\n".post}; ""
