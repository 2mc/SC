BMMultichannelPluginsRack manage multichannel plugins


Inherits from: Object : BMAbstractAudioChainElement


BMMultichannelPluginsRack manages multichannel plugins (instances of BMMultichannelPlugin). It can thus provide a multichannel processing stage at a particular point in an audio chain.


This functions in a similar way to BMTrimPluginsRack and BMTrimPluginsStrip. The primary difference is that there is only a single 'strip' since the channel inputs and outputs for each plugin are arbitrary. Plugins do execute in the specified order however, so it is possible to chain them by using the output channels of one as the inputs of another.


See also: BMMultichannelPlugin, BMMultichannelPluginSpec, BMInOutArray, BMAbstractAudioChainElement, BMMultichannelPluginsRackGUI, BMTrimPluginsRack


Creation / Class Methods


*new (ins, outs, target, addAction, name)

Create a new BMMultichannelPluginsRack.

ins - An instance of BMInOutArray indicating the input channels which are to be processed.

outs - An instance of BMInOutArray indicating the available output channels. If nil is passed here the ins will be used as outputs as well.

target - Target for this object's Group. If target is not a Group or Synth, it will be converted as follows: If it is a Server, it will be converted to the default_group of that server. If it is nil, to the default_group of the default Server. Note: A Synth is not a valid target for \addToHead and \addToTail.

addAction - one of the following Symbols:

\addToHead - add at the head of the group specified by target

\addToTail - (the default) add at the tail of the group specified by target

\addAfter - add immediately after target in its server's node order

\addBefore - add immediately before target in its server's node order

\addReplace - replace target and take its place in its server's node order

name - A Symbol indicating the name of this object. If not supplied one will be generated.



Accessing Instance and Class Variables

plugins

Returns a List of this strip's BMMultichannelPlugin instances in the order in which they execute.

Instance Methods


clear

Remove all plugins from this rack.



mappings

Returns an IdentityDictionary containing the current values for each channel for this rack, in the form IdentityDictionary[plugins->[[plugin.spec.name, plugin.inputs, plugin.outputs, plugin.attributes, plugin.values], ...] ]. Subclasses may override this to include additional parameters.



mappings_ (dict)

Set this objects trim and plugin settings.

dict - An IdentityDictionary containing the current values for each channel for this rack, in the form dentityDictionary[plugins->[[plugin.spec.name, plugin.inputs, plugin.outputs, plugin.attributes, plugin.values], ...] ]. Subclasses may override this to include additional parameters.



addPlugin (plugin)

Add a plugin to this rack, at the end of its chain of plugins.

plugin - A BMMultichannelPlugin.



removePlugin (indexOrPlugin)

Remove a plugin from this rack's chain of plugins.

indexOrPlugin - Either the index of the plugin to remove or the BMMultichannelPlugin itself.



movePluginUp (index)

Move a plugin up one position in this rack's chain of plugins.

index - The index of the plugin to move.



movePluginDown (index)

Move a plugin down one position in this rack's chain of plugins.

index - The index of the plugin to move.



free

Free this strip's plugins, group and other resources. After this you should not use this object.



gui

Creates and returns a BMMultichannelPluginsRackGUI using this as an argument.



Examples


s.boot;


// 5.1 Array (-0.1)

(

l = [[-30, 'L'], [30, 'R'], [0, 'C'], [-110, 'Ls'], [110, 'Rs']].collectAs({|el, i|

BMSpeaker.newFromSpherical(el[1], i, el[0], 0, 2);

}, BMInOutArray);

)


b = Bus.audio(s, 1);


x = {Out.ar(b, PinkNoise.ar(0.2))}.play; // something to pan


i = b.asBMInOutArray(\noiseBus);


r = BMMultichannelPluginsRack(i, l, x, \addAfter, 'Plug em In');


r.addPlugin(BMMultichannelPlugin('2D VBAP Auto Panner', i, l));


s.meter;


r.gui; // play with the GUI


x.free; r.free; b.free;
