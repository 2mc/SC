BMAbstractAudioSource abstract superclass of audio sources


Inherits from: Object : BMAbstractAudioChainElement


BMAbstractAudioSource is the abstract superclass of all BEASTmulch Library objects which serve as sources in an audio chain. It inherits most of its methods from its superclass.


See also: BMAbstractAudioChainElement, BMSoundFilePlayer, Time References and Automation


BMAbstractAudioSources as Time References


Most subclasses of BMAbstractAudioSource should be able to serve as time references, i.e. serve as clock sources for automation, etc. Other objects using instances as references can register as dependants, and thus be updated when changes in time, playing status, etc. occur. Details of how to handle this in subclass implementations are discussed below. See also Time References and Automation.


Subclassing


The following methods should be implemented if this class is going to serve as a time reference. In the discussion below the term 'playback' should be taken to mean advancing the current time for this reference, rather than necessarily referring to a soundfile.


init

In addition to doing any setup and calling initNameAndTarget (see BMAbstractAudioChainElement) this method should include the following:

BMTimeReferences.addReference(this);


play

In addition to any subclass specific implementation needed to start 'playback' and start advancing time, this method should call:

this.changed(\play);



pause

In addition to any subclass specific implementation needed to pause 'playback' and hold at the current time, this method should call:

this.changed(\pause);



stop

In addition to any subclass specific implementation needed to stop 'playback' and reset the current time to 0, this method should call:

this.changed(\stop);

this.changed(\time, 0, 0, Main.elapsedTime);



togglePlay

This method should toggle between playing and pausing.



setTime (time)

This method should jump 'playback' to the specified time, and should call this.changed(\time, 0, playbackRate, Main.elapsedTime) if this will not happen indirectly.
