BMAbstractAudioChainElement abstract superclass of audio chain elements


Inherits from: Object


BMAbstractAudioChainElement is the abstract superclass of all BEASTmulch Library objects which serve as elements in an audio chain. This includes sources (subclasses of BMAbstractAudioSource), such as BMSoundFilePlayer, routers and mixers such as BMAudioMatrix and BMAmpControlMatrix, and processors and spatialisers such as BMTrimPluginsRack and BMMultichannelPluginsRack. As such it defines a standard interface and a number of inherited methods.


Subclasses generally create a Group to hold all of their nodes which is stored in the group instance variable.


For the sake of simplicity and to avoid conflicts subclasses of BMAbstractAudioChain do no rebuilding on Cmd-. If you want to do this simply add a Function to rebuild the chain to ServerTree.


You can see a list of all available subclasses by executing the following:


BMAbstractAudioChainElement.dumpClassSubtree;


See also: BMAbstractMatrix, BMAbstractAudioSource, BMAbstractController, BMAbstractGUI, BMInOutArray


Making chains of elements


Subclasses take instances of BMInOutArray to specify input and/or busses. In some cases output busses will be the same as inputs, in others (notably BMAudioMatrix) take separate outputs. *new methods generally call asBMInOutArray on ins and outs, which most subclasses implement to return their output array. This means that instances can be passed as arguments for these. Some classes (e.g. subclasses of BMAbstractAudioSource) do not take inputs and supply their outputs as a private bus.


asTarget is implemented to return the object's group, which means that you can pass audio chain elements most places you would pass a Node, e.g. as a target for a Synth.


By convention subclasses of BMAbstractAudioSource use \addToHead as their default addAction, and all other subclasses of BMAbstractAudioChainElement use \addToTail. This means that in most cases chains of objects can be created in the order they will appear on the server.


The examples below demonstrate this.


Accessing Instance and Class Variables

*allChainElements

An IdentityDictionary containing all active instances of this class' subclasses. This can be used to query the library to see which chain elements exist at the moment.

addAction

Returns this object's addAction, which determines where its group will be created relative to its target. See subclasses for details.

target

Returns this object's target, which determines the relative position at which its group will be created base on its addAction See subclasses for details.

ins

Returns this object's inputs as a BMInOutArray.

inNames

Returns an Array of Symbols corresponding to the names of this object's ins.

outs

Returns this object's outputs as a BMInOutArray

outNames

Returns an Array of Symbols corresponding to the names of this object's outs.

group

Returns this object's Group if it is active, nil otherwise.

name

Returns this object's name as a Symbol.

server

Returns this object's Server.

Instance Methods and Subclassing


The following methods specify the minimum interface for a BMAbstractAudioChainElement; some notes about subclassing are included. In some cases the methods simply return a SubclassResponsibilityError, and should be overridden in subclasses.


Implementations of *new in subclasses should take at least target, addAction, and name as arguments.


// simplest case

*new { |outs, target, addAction = \addToTail, name| 

^super.new.initNameAndTarget(target, addAction, name);

}

Subclasses which take ins or outs as arguments to *new should implement an init method to copy those to the corresponding instance vars, and fill in inNames and outNames.


// example with ins and outs

*new { |ins, outs, target, addAction = \addToTail, name|

^super.new.init(ins.asBMInOutArray, outs.asBMInOutArray, target, addAction, name);

}

init {|argins, argouts, argTarget, argAddAction, argName|

this.initNameAndTarget(argTarget, argAddAction, argName);

ins = argins;

outs = argouts;

inNames = ins.keys;

outNames = outs.keys;

}

initNameAndTarget (argtarget, argaddAction, argname)

Converts argtarget to a valid target, calls makeName if needed, and adds this object to allChainElements. Subclasses should call this method from their *new or init methods. 

argtarget - Target for this object's Group. If target is not a Group or Synth, it will be converted as follows: If it is a Server, it will be converted to the default_group of that server. If it is nil, to the default_group of the default Server. Note: A Synth is not a valid target for \addToHead and \addToTail.

argaddAction - one of the following Symbols:

\addToHead - (the default for BMAbstractAudioSources) add at the head of the group specified by target

\addToTail - (the default for most subclasses) add at the tail of the group specified by target

\addAfter - add immediately after target in its server's node order

\addBefore - add immediately before target in its server's node order

\addReplace - replace target and take its place in its server's node order

argname - Explanation of argname. Default value is nil. Other information.


// simplest case

initNameAndTarget {|argtarget, argaddAction, argname|

target = argtarget.asTarget;

server = target.server;

addAction = argaddAction;

name = (argname ?? {this.makeName}).asSymbol;

allChainElements[name] = this;

this.makeGroup;

}


gui

This should create and return an appropriate GUI, usually using a specific GUI class (e.g. BMMatrixGUI for matrices) with this as an argument.



mappings_

mappings

These methods should be implemented to allow getting or setting presets, i.e. they should set or get the global state of the object. Usually these return or take some sort of IdentityDictionary. The default implementations do nothing.



makeGroup

Makes this object's Group using target and addAction.



makeName

Automatically generate a name for this object if one has not been supplied.



free

The default implementation frees group and removes this object from allChainElements. Subclasses should override this to do any necessary cleanup of resources. Once called, objects should not be used again.



asTarget

Returns the element's Group. This means that chain elements can be passed directly as targets in most cases. Subclasses can override this to return something else if appropriate.



Examples


// making an audio chain


s.boot;


// make a 5.1 speaker list (-0.1) for outputs

(

l = [[-30, 'L'], [30, 'R'], [0, 'C'], [-110, 'Ls'], [110, 'Rs']].collectAs({|el, i|

BMSpeaker.newFromSpherical(el[1], i, el[0], 0, 2);

}, BMInOutArray);

)


// make a controller

c = BMVirtualController.new('Level Sliders', s, 4); // new controller

g = BMVirtualControllerSliders(c); // GUI sliders for it


//// sources

p = BMSoundFilePlayer(8, name:\foo); // maximum 8 channel soundfile player

p.read("sounds/a11wlk01.wav"); // mono, so only out foo-1

p.gui;


b = Bus.audio(s, 1);

x = {Out.ar(b, PinkNoise.ar(0.2))}.play(target: p, addAction:\addAfter); // a noise source; note p works as a target


// do some processing at this stage

t = BMTrimPluginsRack(p.asBMInOutArray ++ b.asBMInOutArray(\noiseBus), x, \addAfter); // x as target

t.gui;


// make a matrix for connections

m = BMAudioMatrix(t, l, name:'Router'); 


// a matrix to map controls to scale amplitude

a = BMAmpControlMatrix(c, l, name:'Level Control'); // ins are the controller's control busses


m.gui; a.gui; // make some assignments here


// congrats, you've now got sources->trim-plugins->routing->level-scaling->outputs

// turn up your controllers to listen


[p, c, b, x, m, a, t].do(_.free); // cleanup
