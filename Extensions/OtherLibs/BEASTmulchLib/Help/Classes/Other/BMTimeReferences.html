BMTimeReferences library-wide registry for time references


Inherits from: Object


BMTimeReferences provides a central registry and some utility functions for Time References in the BEASTmulch library. Time References are objects which can supply a current time for use with automation, etc.


See also: Time References and Automation


Class Methods


*addReference (ref)

Classes which can serve as time references should call this in their initialisation code (passing themselves as ref) in order to register themselves as an available time references.

ref - The object to be added as a Time Reference.



*removeReference (ref)

Classes which can serve as time references should call this in their cleanup code (passing themselves as ref) in order to unregister themselves as an available time references.

ref - The Time Reference object to be removed.



*timeReferences

Returns a Set containing the names of all available time References as Symbols.

p = BMSoundFilePlayer(name:\foo); // this calls *addReference in its init code

BMTimeReferences.timeReferences; // contains foo

p.free; // free p's resources, calls *removeReference

BMTimeReferences.timeReferences; // foo who?



*currentTime (ref)

Calculate the current time in seconds for the Time Reference named ref. This will be based on the last available reference time and rate of change. This method can be used by classes which may need to update at a different rate than the time reference itself, such as automators.

ref - The object whose time should be calculated.

p = BMSoundFilePlayer(name:\foo);

BMTimeReferences.currentTime(p); // 0

p.read("sounds/a11wlk01.wav");

(

p.play(0, 0).rate_(0.1); // play at 1/10 normal speed

SystemClock.sched(1, { BMTimeReferences.currentTime(p).postln; 1}); // changes slower than normal

)

p.free;
