BMAbstractMIDIController abstract superclass of MIDI controllers in the BM library


Inherits from: Object : BMAbstractController


BMAbstractMIDIController is the abstract superclass of generic MIDI controllers in the BEASTmulch library.


See also: BMAbstractController, BMMIDICCController, BMMIDIBendController, BMControl



Accessing Instance and Class Variables

acceptsAutomation_(bool)

acceptsAutomation

By default this is false, but can be set to true. Some MIDI controllers are motorised for instance, and thus automatable.

loopBack_(bool)

loopBack

Get or set a Boolean indicating whether incoming messages are echoed back out to the controller. Some more complex MIDI devices require this. The default is false.

midiout

Get this controller's MIDIOut object.

Subclassing


The following methods (at least) must be implemented by subclasses in addition to those outlined in BMAbstractController.

startListening

This should add appropriate responder object(s) to update values when messages are received. It should call updateValue to do this.

makeInputSpec

This should create a spec to convert from incoming values to 0..1.

// sample implementation from BMMIDICCController

makeInputSpec {

spec = [0, 127].asSpec;

}



setNumControls

This should set the numControls instance var appropriately.

loopback (ind, val)

This should send incoming messages back to the controller if needed.


Instance Methods


updateValue (ind, val)

Called by responders when a MIDI message is received to update control busses and call loopback if specified.

ind - Index of control to update. Starts from 0.

val - Value to update. This should be in native units (e.g. 0..127) rather than 0..1.



updateAllValues (array)

Update all values at once.

array - An array of values to update. These should be in native units (e.g. 0..127) rather than 0..1.
