BMArbitraryStartSnapShot an arbitrary intial state in an automation sequence


Inherits from: Object : BMAbstractSnapShot


BMArbitraryStartSnapShot represents an unknown initial state in a snapshot automation sequence. For example, if dealing with a motorised fader board, it may not be possible to know the state of the faders in advance of starting the sequence. Placing a BMArbitraySnapshot at the start of an automation sequence indicates that the intial state should be sampled when the sequence becomes active, and those values used as the start of the first segment. This means that an automation sequence can interpolate from an unknown to a known state, and allows users to combine live control with automation.


Under normal circumstances you will not need to make these directly. A BMArbitraryStartSnapShot is automatically created as the first snapshot of a BMSnapShotSeq.


See also: BMSnapShot, BMAbstractSnapShot, BMSnapShotSeq, BMAbstractAutomator, BMControl, BMAbstractController, Time References and Automation


Creation / Class Methods


*new (controls, time, name)

Make a new arbitrary snapshot.

controls - An Array of Symbols corresponding to the names of the BMControl instances to be automated.

time - Time in seconds for this snapshot. This will be relative to a Time Reference. See Time References and Automation.

name - A Symbol indicating the name of this snapshot.



Accessing Instance and Class Variables

snapTime

Returns the time at which the controller state was determined, or nil if this segment is inactive. This will correspond to the moment at which the automator first called automate at a time equal to or later than this snapshot's time.

Instance Methods


isKnown

Returns false as this is an arbitrary snapshot.
