BMAbstractSnapShot abstract superclass of automation snapshots


Inherits from: Object


This is the abstract superclass of automation snapshots. Snapshots represent the momentary state of something being automated.


See also: BMSnapShot, BMArbitraryStartSnapShot, BMSnapShotSeq, BMAbstractAutomator, BMControl, BMAbstractController, Time References and Automation


Creation / Class Methods


*new (controls, time, name)

Make a new snapshot.

controls - An Array of Symbols corresponding the names of the BMControl instances to be automated.

time - Time in seconds for this snapshot. This will be relative to a Time Reference. See Time References and Automation.

name - A Symbol indicating the name of this snapshot.



Accessing Instance and Class Variables

values

Returns an IdentityDictionary in the form of [ctrlname1->value1, ctrlname2->value2, ...] indicating the values of the automated controls at this snapshot's time.

time

Returns the time in seconds for this snapshot. This will be relative to a Time Reference. See Time References and Automation.

name

Returns a Symbol indicating the name of this snapshot.

controlNames

Returns an Array of Symbols corresponding the names of the BMControl instances being automated.

Instance Methods


time_ (newTime)

Set the time of this snapshot.

newTime - The new time in seconds.



isKnown

Returns a Boolean indicating if this snapshot is arbitrary. Arbitrary snapshots are those whose state can only be determined when they become active in an interpolation segment. The default implementation returns true since most snapshots are not arbitrary. Subclasses can override this to return false if need be. See BMArbitraryStartSnapShot for more information.



setValue (ctrl, value)

Sets the value for a particular control.

ctrl - A Symbol indicating the name of the control.

value - The new value, usually a Float.
