BMAbstractAutomator abstract superclass of all automators


Inherits from: Object


This is the abstract superclass of all automators within the BEASTmulch library. All automators are dependants of a time reference (i.e. a clock source), which updates the automator through a changed message which includes arguments for the updated time, a rate of change, and a reference time indicating at what moment the updated time was accurate.


Automators should respond without failing to variable rates of time change (including backwards), and to time values of nil, < 0, and +/- infinity.


See also: BMControllerAutomator, BMAbstractIndependentRateAutomator, Time References and Automation



Accessing Instance and Class Variables

timeReference

timeReference_

Returns or sets the object which functions as this automator's time reference (i.e. its clock source). This might be any object which complies with the guidelines set out in Time References and Automation.

running

Returns a Boolean indicating whether this automator is currently running.

Subclassing


Subclasses should implement at least these methods.


automate

This method will be called in some sort of loop every time automation needs to be updated. When and how often this occurs can be subclass dependent. This is where the actual updates to whatever is being automated should occur.



mappings

Returns the subclass specific objects (e.g. automation sequences) which correspond to the automator's current settings. This should be global.



mappings_

Sets the subclass specific objects (e.g. automation sequences) which correspond to the automator's current settings. This should be global, so setting mappings overwrites existing settings.



update

This must respond appropriately (perhaps doing nothing) to at least the following changed messages from the time reference: \time, \stop, and \base. See Time References and Automation for more detail.


Instance Methods



timeInitialised

Returns a Boolean indicating whether or not this automator's time, rate, and referenceTime have been initialised.
