BMAbstractIndependentRateAutomator independent update rate automator


Inherits from: Object : BMAbstractAutomator


BMAbstractIndependentRateAutomator is the abstract superclass of automators that have an update rate which is independent from that of its time reference. In some cases it can be useful or necessary to automate at a separate rate, for instance when automating a GUI, which may cause hangs with a fast update rate.


If the automation rate is different from that of the time reference a best guess of the current time is made using the last available rate and reference time.


See also: BMControllerAutomator, BMAbstractAutomator, Time References and Automation, BMTimeReferences


Accessing Instance and Class Variables

interval_(rate)

interval

Get or set the rate in seconds at which automation is updated.

Subclassing


The following methods may need to be overrriden in subclasses. See also BMAbstractAutomator.

startUpdateLoop

This sets running to true and starts a while loop calling automate at this automator's interval. The default implementation is as follows, but you can override to add any necessary initialisation:

startUpdateLoop {

running = true;

Routine({

while({running}, {

this.automate;

interval.wait;

});

}).play;

}



stopUpdateLoop

This stops the update loop and thus ends automation. The default implementation simply sets running to false, but you can override to add any necessary cleanup code.



reset

This is called when the time reference sends a changed: \stop to the automator. This resets the automator, and does any necessary cleanup. The default implementation does nothing.
