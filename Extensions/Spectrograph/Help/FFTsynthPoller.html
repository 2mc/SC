FFTsynthPoller


Inherits from: AbstractUniqueServerObject : UniqueObject : Object

 

Manages a FFT analysis process, polls the data from that process at a steady rate, and distributes the polled data array to any "listener" client that wants to do something with them. Is used by Spectrograph. 

 

FFTsynthPoller is created and used by Spectrograph. Other objects can receive its data via the update message, by adding themselves as Listeners using anFFTsynthPoller.addListener(anObject); 


Creating an FFTsynthPoller and receiving data from it


// One by one: 

//:Create, start and set the rate

FFTsynthPoller(\test).start.rate = 0.210;


//:Add the listener Function (can also be an object using the update method to receive data)

FFTsynthPoller.at(\test).addListener({ | ... data | data.postln });


//:Stop the poller

FFTsynthPoller(\test).stop;


// =====================================================================


//:More succinctly. Also removing the listener: 

f = { | ... data | data.postln }; // must store the function to be able to remove it as object

FFTsynthPoller(\test).start.rate_(0.210).addListener(f);


//:Remove the listener

FFTsynthPoller(\test).removeListener(f);


//:Stop the poller

FFTsynthPoller(\test).stop;


//:-


Example of controlling FFTsynthPoller's rate.  


Note: This is also the method used by Spectrograph to set the rate. 

See Spectrograph:start and Spectrograph:rate_  


//:Start a Spectrograph

Spectrograph.small;


//:Once it is running, access the FFTsynthPoller to add a message

// Note: the sender: \test does not have to be a Symbol but can be any Object.

FFTsynthPoller.at('Spectrograph:localhost').addMessage(\test, \rate_); 


//:test sound 

\default.play({ SinOsc.ar(LFNoise0.kr(20).range(100, LFSaw.kr(0.1, pi).range(100, 12000)), 0, 0.05) });


//:Change the rate through NotificationCenter. 

// Note: any object can send

 

\test.notify(\rate_, 0.5);


//:change rate back to faster:

\test.notify(\rate_, 0.04);
