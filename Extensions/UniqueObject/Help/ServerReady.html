Send notifications after a Server has called initTree and created its root node, but only when it boots. 


ServerReady ensures that any SynthDefs created as Udef instances or any Buffers created as UniqueBuffer instances will be loaded to the buffer before any synth starts. This is ensured when using the classes

Udef, UniqueBuffer, UniqueSynth, UniquePlay either directly or by sending the messages play, playDef, playFunc on Symbols or the messages uplay, mplay on Functions (see Shortcuts.html). 


Additionally, any function can be added to be evaluated after the Server has booted and all SynthDefs and Buffers are loaded, using ServerReady.addAction(action, server), or ServerReady.addObjectAction(object, action, server);



General User Interface


*addAction(action, server) Evaluate action ONCE when the server is ready 


action: a Function 

server: a server to monitor for booting. Default is Server.default


Perform an action ONCE ONLY when the server is ready, i.e. when 

all synthdefs and buffers have been loaded.

If the server is already running and ready, the action will be executed immediately. 




*addObjectAction(object, action, server) Evaluate action for object AT EACH BOOT. 

Remains active only as long as the object has not received the objectClosed 

or remove messages. 


object: any Object

action: a Function 

server: a server to monitor for booting. Default is Server.default


Perform an action for object when the server is ready, i.e.  

when all synthdefs and buffers have been loaded.

Always repeat the action, whenever the server boots, as long as 

the object or the ServerAction instance have not received the objectClosed

or remove message. 

Clean up notification connection when either ServerReady or the 

object receive the objectClosed or removed message.

See Object:objectClosed, UniqueObject.remove.
